hoisting 的原理

前言

正常來說，程式碼執行的順序是由上往下，
但是所謂的 提升，就不是按照這樣的順序來走，
被提升的程式碼，我們可以想像它們是被拉到第一行的位置了，
既然是在第一行，意味著底下的任何地方都能被存取到 hoisting 的函式或變數。

let, const, var 跟函式都有 提升 (hoisting) 現象，
但 hoisting 指的是部分的程式碼產生的現象，

共通點是，宣告的變數會提升，但賦值不會。


拆解 var 的提升

因為 var 的宣告會提升，賦值不會，
在賦予值之前的 a 是 undefined。
這也解釋了兩個例子都得到同樣的結果 undefined。


拆解 function 的提升

如果是變數，要先被宣告之後才能使用，
但 function 可以先被呼叫，後面再宣告，
原因也是 hoisting，所以在任何地方宣告函式都無所謂！

兩種函式宣告的方法：

  • 函式一般宣告
  • 函式變數宣告

函式一般宣告

可以視為是整個函式都一起被提升了，
因此函式不需要移到最上方才能被呼叫，是很方便的設計。


函式變數宣告

用變數宣告的函式，會很像在宣告變數，
也就是宣告的部分有 hoisting，但賦值的部分沒有，
因此第 2 行我們得到 “TypeError: sayHi is not a function”，
原因是用函式的呼叫方法呼叫它，
實際上它的值(函式)沒有一起被提升，它在第 2 行還不是個函式！


函式內部的 hoisting

在函式作用域內的變數，也有 hoisting 現象，規則是：

函式作用域內的變數 > 函式外部的變數


在函式內若同時出現變數、參數與函式，其提升的優先順序為：

函式 > 參數 > 變數


let & const：TDZ 暫時死區

「暫時死區」的白話文：

let 和 const 宣告的變數之後的 hoisting 的現象，
並持續到它被賦予值之前的這個「空檔」。

由於 let 和 const 宣告的變數也會提升到區塊作用域，
如果在宣告之前使用這個變數，你會看到 dev tool 出現：
也就是提醒你，不能在未初始化前，使用這個變數！

Uncaught ReferenceError: Cannot access ‘test’ before initialization
