switch 和 if 的差異和使用時機
Karen Huang
  2021-12-16 21:40:17 2021-12-16 21:40  
  • JavaScript 
  • > 流程判斷 
 
  • JavaScript 
 

這一篇主要在探討 JavaScript 中的 switch (switch-case) 和 if 判斷式(if-else statement) 之間有什麼異同，
又應該如何判斷選用的時機？

比較 switch 和 if 的運算過程

首先，兩者在外觀上就有很大的差異，因此本文大致上是：

  1. 比較兩者語法的異同。
  2. 最後找出各自的強項，分析出適合選用的時機。

(一) 強制轉型

switch 不允許強制轉型

switch 的預設機制就是 === 的判斷，直接比對變數 number 跟 case 的資料型別、再比較值，
所以若是型別不同的將會第一時間被篩選掉，因此在底下 switch 範例中的 case ‘666’ 不會執行。

if 判斷式允許強制轉型，但可透過運用 === 避免被轉型：

if 判斷式 ( ) 內的運算，若不希望被 JS 隱性轉型，要另外透過用 ===、!==、>== ，
指示運算過程不允許強制轉型，否則就會出現以下的狀況，明明答案要是數字666，卻印出字串666。

(二) 範圍取值

switch 不擅於範圍取值

相較之下，switch 就不太擅於進行範圍取值，若勉強要進行，
它也不像 if 判斷式能運用到邏輯運算子來算出一個區間範圍（像是：介於…到…之間），
頂多能做到以下範例中的比對，但記得要在 switch 的變數直接改為 true。
相對來說，switch 還是在值與值之間的比對顯得會更加精準跟快速。

if 判斷式較為擅長進行範圍取值

如同前面所提過，if 判斷式可透過比較運算子或邏輯運算子得出 true 或 false。

(三) switch 沒有 break 就不會煞車

在 switch 的程式碼中，沒有 break 來終止程式碼運行，就算已經有比對到 true 的結果，後方的 case 仍會繼續跑完；
而 if 判斷式在比對到 true 的結果後，就會終止運算。

(四) default：當上述條件都不符合時，就執行

(五) switch 可以沒有 default，如同 if 判斷式也可以沒有 else

總結

在看完以上的比較之後，發現 if 判斷式和 switch 之間，其實有很多相同和不同之處。
嚴格真的要比較起來，其實沒有誰好壞之分，但可以說它們各有不同擅長的功能，
在了解兩者的特性後，可依照自己的需求，在不同的使用時機下選用。

switch 的運用時機

  • 比對一個變數的值和另一個值的簡單判斷，switch 在效能上有效率。
  • switch 可以直接比對變數跟 case 後方的值。
  • switch 強制不允許資料轉型，運算上更為嚴謹。

if 的運用時機

  • 能判斷 true 或 false。
  • 能判斷非布林值的值。
  • 能彈性決定是否強制轉型（switch 不允許資料轉型）。
